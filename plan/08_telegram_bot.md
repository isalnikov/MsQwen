# Веха 8: Telegram бот

## Связь с C4 моделями
- **Level 2 (Containers)**: Telegram Bot — отдельный контейнер в системе
- **Level 3 (Components)**: Компоненты бота (команды, клавиатуры, диалоги)
- **Sequence Diagrams**: Взаимодействие пользователя с ботом

## Цель
Реализовать интерактивного Telegram бота с командами для управления системой, respecting user isolation и supporting all metadata display согласно C4 моделям.

## Важные принципы
- **Изоляция данных**: Бот показывает только данные текущего пользователя
- **Все метрики**: views, forwards, reactions, likes, engagement_score доступны для отображения
- **Inline клавиатуры**: Для интерактивных команд
- **State management**: Для многошаговых диалогов
- **Уведомления**: О завершении парсинга и анализа
- **Java 25+**, Virtual Threads где применимо
- **Без Lombok**, чистый код

### 8.2 Команды бота

**ВАЖНО**: Все команды показывают только данные текущего пользователя.

#### Базовые команды
- [ ] `/start` - Приветствие и главное меню
  - Регистрация пользователя если новый
  - Кнопки: "Мои промпты", "Новости", "Анализ", "Помощь"
- [ ] `/help` - Справка по всем командам
- [ ] `/version` - Информация о сборке (версия, время, билд)
- [ ] `/status` - Статистика системы
  - Кол-во новостей, промптов, каналов пользователя
  - Кол-во проанализированных/непроанализированных новостей

#### Команды промптов
- [ ] `/prompts` - Список активных промптов с кнопками действий
  - Inline клавиатура с выбором промпта
- [ ] `/addprompt` - Добавить новый промпт
  - Диалог ввода названия и текста промпта
  - Сохранение с userId
- [ ] `/editprompt` - Редактировать промпт
  - Выбор из списка промптов пользователя
- [ ] `/deleteprompt` - Удалить промпт
  - **Предупреждение**: Удаляются все каналы, новости, результаты анализа
  - Подтверждение удаления

#### Команды каналов
- [ ] `/channels` - Список каналов выбранного промпта
  - Выбор промпта через inline клавиатуру
- [ ] `/addchannel` - Добавить канал
  - Диалог ввода handle, URL, описания
  - Привязка к промпту и пользователю
- [ ] `/deletechannel` - Удалить канал
  - **Предупреждение**: Удаляются все новости канала
  - Подтверждение удаления

#### Команды новостей
- [ ] `/parse` - Запустить парсинг всех каналов
  - Парсинг каналов пользователя
  - Уведомление о начале/окончании
  - Статистика: сколько новостей добавлено
- [ ] `/analyze` - Запустить анализ собранных новостей
  - Выбор промпта (если несколько)
  - Отправка в Qwen CLI со всеми метаданными
  - Уведомление о завершении анализа
  - Результат анализа отправляется пользователю
- [ ] `/news` - Получить новости/результаты анализа
  - Пагинация если новостей много
  - Кеширование (если кеш пуст - свежий анализ)
  - Показ всех метрик: views, forwards, reactions, likes, engagement_score

#### Админ команды
- [ ] `/clearnews` - Очистить базу новостей (требует подтверждения, admin only)
- [ ] `/clearall` - Полная очистка данных (только admin)
