# Веха 9: CLI интерфейс (Command Line Interface)

## Связь с C4 моделями
- **Level 2 (Containers)**: CLI — контейнер для администрирования
- **Level 3 (Components)**: CLI компоненты командной строки

## Цель
Реализовать консольный интерфейс для администрирования и запуска задач с параметрами командной строки согласно C4 моделям.

## Важные принципы
- **CommandLineRunner**: Spring Boot CLI режим
- **Параметры запуска**: Токены, каналы, настройки планировщика
- **Изоляция данных**: CLI может работать с данными конкретных пользователей
- **Админ операции**: Очистка таблиц (news, channels, users)
- **Логирование**: Консоль + файл, без System.out.println()
- **Java 25+**, Virtual Threads где применимо

### 9.3 Параметры запуска

**ВАЖНО**: Все параметры могут быть заданы через аргументы командной строки или application.yml.

- [ ] `--telegram.token=<token>` - Токен бота
- [ ] `--telegram.chat-id=<id>` - Chat ID для уведомлений
- [ ] `--qwen.api-key=<key>` - API ключ Qwen (если нужен)
- [ ] `--scheduler.enabled=<true|false>` - Включить планировщик
- [ ] `--parser.channels=<channel1,channel2>` - Каналы для парсинга
- [ ] `--prompt.id=<id>` - ID промпта для анализа
- [ ] `--user.id=<id>` - ID пользователя для операций
