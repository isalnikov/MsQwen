## Предварительные требования (верификация стека)

### Java и Spring Boot
- [ ] Подтверждено: проект использует **Java 25+** (проверить `java -version` и `pom.xml` `<java.version>`)
- [ ] Подтверждено: проект использует **Spring Boot 4+** (проверить `pom.xml` `<spring-boot.version>`)
- [ ] Подтверждено: **Lombok НЕ используется** в проекте — все геттеры/сеттеры/конструкторы написаны вручную
- [ ] Подтверждено: все современные фичи Java применяются где уместно:
  - `record` для DTO
  - `var` для локальных переменных
  - `switch expressions`
  - `text blocks` для многострочных строк (промпты, SQL)
  - `Pattern Matching for instanceof`
  - `Sequenced Collections`
  - `Virtual Threads` (где применимо)

### Чистота кода
- [ ] Нет `System.out.println()` — используется только `Logger`
- [ ] Логирование во всех классах через `private static final Logger logger = LoggerFactory.getLogger(ClassName.class);`
- [ ] `@Autowired` используется **только в тестах**, в основном коде — внедрение через конструктор (`final` поля)
- [ ] Каждый класс имеет одну ответственность (SOLID)
- [ ] Нет дублирования кода (DRY)
- [ ] Нет излишней функциональности (YAGNI)
- [ ] Код простой и понятный (KISS)
- [ ] **Javadoc 100% покрытие** — каждый класс, метод, поле с публичным API имеет документацию на русском языке

### 13.4 База данных (ПРИОРИТЕТ #1)
- [ ] **Liquibase миграции созданы** (V1-V5 или больше) — каждый скрипт идемпотентный

- [ ] **JPA Entity классы созданы без Lombok** — все геттеры/сеттеры/конструкторы/equals/hashCode/toString вручную
- [ ] **Тесты на каскадное удаление** — проверить что при удалении Prompt удаляются связанные Channel и News
- [ ] Очистка таблиц реализована (CLI + API `/api/v1/cleanup`)
- [ ] Автоматическое удаление новостей старше 30 дней по расписанию

### 13.10 REST API
- [ ] Все endpoints реализованы с префиксом `/api/v1`:
  - `POST /users/register` — Регистрация пользователя
  - `GET /users/me` — Получить текущего пользователя

  - `GET /prompts` — Список промптов пользователя

  - `GET /channels/{id}` — Список каналов по промпту

  - `POST /parse` — Запустить парсинг
  - `POST /analyze` — Запустить анализ
  - `GET /news/{id}` — Новости по промпту
  - `GET /analysis/{id}` — Результат анализа
  - `DELETE /cleanup` — Очистка всех таблиц (только для консоли/админа)
- [ ] **Swagger/OpenAPI документация** доступна (Swagger UI)
