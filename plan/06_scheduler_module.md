# Веха 6: Планировщик задач (Scheduler)

## Связь с C4 моделями
- **Level 2 (Containers)**: Планировщик — часть контейнера Spring Boot
- **Level 3 (Components)**: Scheduler компоненты показывают задачи
- **Data Flow**: Автоматические потоки данных по расписанию

## Цель
Реализовать автоматический парсинг и анализ новостей по расписанию с использованием Virtual Threads (Java 25+) согласно C4 моделям.

## Важные принципы
- **Virtual Threads**: Использовать для конкурентного парсинга каналов
- **Очистка старых новостей**: Удалять новости старше 30 дней
- **Обработка ошибок**: Ошибки в scheduled задачах не должны ломать приложение
- **Настраиваемость**: Включение/выключение через application.yml

### 6.2 Scheduled задачи

#### NewsParsingScheduler
- [ ] `@Scheduled(cron = "${scheduler.parse.cron:0 */30 * * * *}")`
- [ ] Метод `parseAllActiveChannels()`
  - Парсинг всех активных каналов всех пользователей
  - Использование Virtual Threads для конкурентного парсинга
- [ ] Сохранение новых новостей в БД
  - Проверка на дубликаты (по message_id + channel_id)
  - Сохранение всех метаданных (views, forwards, reactions, likes, engagement_score)
- [ ] Логирование результатов
  - Количество распарсенных новостей
  - Ошибки парсинга
- [ ] Обработка ошибок
  - Пропуск недоступных каналов
  - Логирование и продолжение

#### NewsAnalysisScheduler
- [ ] `@Scheduled(cron = "${scheduler.analyze.cron:0 0 * * * *}")`
- [ ] Метод `analyzeNewNews()`
  - Выборка непроанализированных новостей (is_analyzed = false)
  - Приоритизация по engagement score (новости с более высокими метриками первыми)
- [ ] Отправка в Qwen CLI для анализа
  - Формирование промпта со всеми метаданными
  - Передача views, forwards, reactions, likes, engagement_score
- [ ] Сохранение результатов анализа
  - AnalysisResult с news_ids, cache_key, TTL
- [ ] Обновление статуса `is_analyzed = true`
- [ ] Отправка уведомлений клиентам (Telegram bot)
- [ ] Обработка ошибок
  - Ошибки Qwen CLI не ломают задачу
  - Логирование неудачных анализов

#### NewsCleanupScheduler
- [ ] `@Scheduled(cron = "${scheduler.cleanup.cron:0 0 3 * * *}")`
- [ ] Метод `deleteOldNews()`
  - Удаление новостей старше 30 дней
  - Логирование количества удаленных записей
  - CASCADE: новости удаляются вместе с связями
- [ ] Обработка ошибок
  - Транзакционность операции
  - Логирование результатов

#### CacheCleanupScheduler
- [ ] `@Scheduled(cron = "${scheduler.cache.cron:0 0 4 * * *}")`
